# Blur Engine — Blur Modes

The engine supports multiple visually distinct blur modes. This document details each mode's mechanism, the CSS it generates, how it handles different element types (text vs. media), and the reveal interaction for each mode.

---

## Mode Enums

Three separate enums cover different blur contexts:

**`blsi.blur_modes`** — blur-all (page-wide) modes:
```js
blur_modes = {
  gaussian: 'gaussian',
  frosted:  'frosted',
  redacted: 'redacted',
  censored: 'censored',
}
```

**`blsi.pick_blur_modes`** — pick-and-blur modes:
```js
pick_blur_modes = {
  blur:    'blur',    // gaussian
  frosted: 'frosted',
  color:   'color',
}
```
Note: `redacted` and `censored` are not available for pick-blur. Pick-blur only offers gaussian, frosted, and solid-color cover.

**`blsi.pii_modes`** — PII detection modes:
```js
pii_modes = {
  blur:     'blur',    // gaussian, fixed 12px
  frosted:  'frosted',
  redacted: 'redacted',
  starred:  'starred',
}
```

---

## Gaussian (Default) Mode

### How it works

Applies `filter: blur(var(--bl-si-radius, 10px))` via CSS. The radius is a CSS custom property set on `:root` by `content_script.applySettingsToDom()`. Changing the radius requires no DOM work from the engine — the CSS engine re-paints with the new variable value at the next repaint.

### CSS generated by `injectRules` (gaussian):

```css
/* Font injection: none */

/* Always-blur tags */
h1:not(...EXCLUDE):not([data-bl-si-pick-blur]):not([data-bl-si-pii]),
h2:not(...)...,
... {
  filter: blur(var(--bl-si-radius, 10px)) !important;
  transition: filter var(--bl-si-transition-duration, 150ms) ease !important;
  user-select: none !important;
}

/* Data-attribute rule for text-check elements */
[data-bl-si-blur]:not([data-bl-si-reveal]) {
  filter: blur(var(--bl-si-radius, 10px)) !important;
  transition: filter var(--bl-si-transition-duration, 150ms) ease !important;
  user-select: none !important;
}

/* Reveal overrides (see 01-css-layer.md for cascade explanation) */
[data-bl-si-reveal] { filter: none !important; ... }
[data-bl-si-reveal] [data-bl-si-blur] { filter: none !important; ... }
...
```

### Media elements

Media elements (`<img>`, `<video>`, `<canvas>`, `<svg>`) are covered by the always-blur tag rule. Gaussian `filter: blur()` applies correctly — it blurs the rendered output of the element, not the element's pixel data (so DRM-protected video is supported).

### Reveal

`filter: none !important` clears gaussian blur. `visibility: visible !important` is a no-op but included for completeness (gaussian doesn't set visibility).

### Reconcile key behavior

`blur_radius` is **excluded** from the reconcile key for gaussian mode:
```js
const reconcileKey = isActive
  ? `${settings.blur_mode}|${JSON.stringify(settings.blur_categories)}|${settings.thorough_blur}|${settings.blur_mode === blsi.blur_modes.frosted ? settings.blur_radius : ''}`
  : 'inactive';
```

This means changing the blur radius in gaussian mode does NOT trigger a page-wide DOM rescan. The CSS var update via `applySettingsToDom()` is sufficient.

## Frosted Glass Mode

### How it works

Applies a CSS SVG filter that creates a frosted glass visual effect — a combination of Perlin noise distortion and Gaussian blur that simulates looking through textured glass.

The SVG filter is injected into the DOM by `ensureSvgFilter(root)`. The CSS rule references it via `url(#bl-si-frosted-filter)`.

### SVG Filter Structure

```js
function ensureSvgFilter(root) {
  const container = (root && root !== document) ? root : document.body;
  if (!container) return;

  // Always rebuild — Chrome's filter cache is not reliably invalidated by
  // mutating feGaussianBlur.stdDeviation in place. A fresh element = fresh cache.
  const existing = container.querySelector('#' + SVG_FILTER_ID);
  if (existing && existing.parentNode) existing.parentNode.removeChild(existing);

  const svgNS = "http://www.w3.org/2000/svg";
  const svg = document.createElementNS(svgNS, "svg");
  svg.setAttribute("id", SVG_FILTER_ID);
  svg.setAttribute("style", "position:absolute;width:0;height:0");  // invisible

  const filter = document.createElementNS(svgNS, "filter");
  filter.setAttribute("id", "bl-si-frosted-filter");

  const turbulence = document.createElementNS(svgNS, "feTurbulence");
  turbulence.setAttribute("type", "turbulence");
  turbulence.setAttribute("baseFrequency", "0.04");  // controls noise scale
  turbulence.setAttribute("numOctaves", "3");          // layering for natural texture
  turbulence.setAttribute("result", "noise");

  const displacement = document.createElementNS(svgNS, "feDisplacementMap");
  displacement.setAttribute("in", "SourceGraphic");
  displacement.setAttribute("in2", "noise");
  displacement.setAttribute("scale", "12");            // distortion intensity
  displacement.setAttribute("xChannelSelector", "R");  // X distortion from red channel
  displacement.setAttribute("yChannelSelector", "G");  // Y distortion from green channel

  const gaussianBlur = document.createElementNS(svgNS, "feGaussianBlur");
  gaussianBlur.setAttribute("stdDeviation", String(_readCssRadius() || 4));

  filter.appendChild(turbulence);
  filter.appendChild(displacement);
  filter.appendChild(gaussianBlur);
  svg.appendChild(filter);
  container.appendChild(svg);
}
```

**Filter pipeline:**
1. `feTurbulence` → generates Perlin noise (random pattern) at `baseFrequency=0.04`
2. `feDisplacementMap` → distorts the source graphic using the noise as a displacement map (scale=12 pixels)
3. `feGaussianBlur` → applies a Gaussian blur with `stdDeviation` from the CSS var

The result is a blurry, slightly warped image — the visual signature of frosted glass.

**Why always rebuild the SVG element:** Chrome internally caches SVG filter results keyed to the filter element's identity. Mutating `feGaussianBlur.stdDeviation` in place does not reliably invalidate this cache in all Chrome versions. Destroying and recreating the element guarantees a fresh cache entry, ensuring radius changes take effect immediately.

**Scope-aware injection:**
```js
const container = (root && root !== document) ? root : document.body;
```
For the main document, the SVG is appended to `<body>`. For a shadow root, the SVG is appended to the shadow root itself. This is critical because CSS `url(#id)` references are resolved within the local stylesheet's scope — a shadow root's CSS cannot reference a `<filter>` element in the main document.

**Position absolute, width/height 0:** The SVG element is invisible and takes up no space. It is a DOM artifact that exists only to define the filter that CSS references.

### CSS generated by `injectRules` (frosted):

```css
/* SVG filter injected separately by ensureSvgFilter() */

h1:not(...EXCLUDE)... {
  filter: url(#bl-si-frosted-filter) !important;
  transition: filter var(--bl-si-transition-duration, 150ms) ease !important;
  user-select: none !important;
}

[data-bl-si-blur]:not([data-bl-si-reveal]) {
  filter: url(#bl-si-frosted-filter) !important;
  transition: filter var(--bl-si-transition-duration, 150ms) ease !important;
  user-select: none !important;
}

[data-bl-si-reveal] { filter: none !important; ... }
...
```

### Reconcile key behavior

`blur_radius` **is included** in the reconcile key for frosted mode:
```js
settings.blur_mode === blsi.blur_modes.frosted ? settings.blur_radius : ''
```

When radius changes in frosted mode, the reconcile key changes, triggering `handleMainDocument()` which calls `injectRules()` which calls `ensureSvgFilter()` — rebuilding the filter with the new `stdDeviation`. Without this, radius changes in frosted mode would be silently ignored.

### Reveal

`filter: none !important` clears the frosted filter. The SVG element remains in the DOM (it's just a filter definition, not a visible element).

## Redacted Mode

### How it works

Replaces the visual content with a solid color rectangle. Text becomes invisible (color: transparent), background fills with the redaction color. Images and videos are hidden via `visibility: hidden`.

### CSS generated by `injectRules` (redacted):

```css
h1:not(...EXCLUDE)...,
[data-bl-si-blur]:not([data-bl-si-reveal]) {
  background-color: var(--bl-si-redaction-color, #000) !important;
  color: transparent !important;
  border-color: var(--bl-si-redaction-color, #000) !important;
  text-decoration-color: transparent !important;
  filter: none !important;      /* cancels gaussian fallback in content.css */
  user-select: none !important;
}
```

**Why `filter: none` is explicit:** The static `content.css` has `[data-bl-si-blur]:not([data-bl-si-reveal]) { filter: blur(...) }`. In redacted mode, this gaussian rule would still apply to `[data-bl-si-blur]` elements unless explicitly overridden. `filter: none !important` in the injected rule wins (same or higher specificity, later source order) and cancels the gaussian.

**Why `border-color` and `text-decoration-color`:** Some elements have visible borders and underlines. Setting them to the redaction color prevents telltale lines that reveal the element's presence.

### Media elements in redacted mode

```css
img[data-bl-si-blur]:not([data-bl-si-reveal]),
video[data-bl-si-blur]:not([data-bl-si-reveal]),
canvas[data-bl-si-blur]:not([data-bl-si-reveal]),
svg[data-bl-si-blur]:not([data-bl-si-reveal]),
picture[data-bl-si-blur]:not([data-bl-si-reveal]),
audio[data-bl-si-blur]:not([data-bl-si-reveal]) {
  visibility: hidden !important;
  user-select: none !important;
}
```

**Why `visibility: hidden` instead of `background-color`:** Images/videos/canvases render pixels, not text. `color: transparent` has no effect on pixel content. `background-color` would show behind the image only if the image is transparent or smaller than its box. `visibility: hidden` hides the element entirely while keeping its layout box (the surrounding context remains stable). The redaction `background-color` from `blurDecl` shows through because the element itself is invisible.

Also applies to always-blur media tags via the tag selector path:
```css
img:not(...EXCLUDE),
video:not(...EXCLUDE),
... {
  /* from blurDecl */
  background-color: var(--bl-si-redaction-color, #000) !important;
  ...
  visibility: hidden !important;
}
```

### Redaction color source

```js
blurDecl = `background-color: var(--bl-si-redaction-color, #000) !important; ...`
```

The CSS var `--bl-si-redaction-color` is set on `:root` by `content_script.applySettingsToDom()`. Changing the redaction color in settings propagates to all redacted elements instantly via the CSS var, without any DOM work from the engine.

For PII redacted mode, the color can be passed as an argument and baked directly into the CSS rule:
```js
const c = (color && /^#[0-9a-fA-F]{6}$/.test(color)) ? color : 'var(--bl-si-redaction-color, #000)';
```

### Reveal

```css
[data-bl-si-reveal] {
  filter: none !important;
  visibility: visible !important;  /* un-hides hidden media */
  font-family: unset !important;
  user-select: auto !important;
}
```

`visibility: visible !important` is critical for redacted mode — it re-shows media elements that were hidden with `visibility: hidden`. Without it, media elements would remain invisible even after reveal.

Background color is NOT explicitly reset in the reveal rule for blur-all redacted mode (the element itself gets `filter: none` which clears filter, but background-color remains until the element loses `[data-bl-si-reveal]`). In practice, reveal is temporary — hover or click — so the brief background color mismatch is acceptable.

## Censored Mode

### How it works

Replaces every text character with a filled disc glyph (●) using a custom WOFF2 font. The font maps every Unicode codepoint to the same filled disc glyph. This is a font-replacement strategy — the text layout reflows to the glyph shape but the underlying text is visually replaced.

Images and videos are blackened with `filter: brightness(0)`.

### Font

`blsi.Fonts.DISC_FONT_FACE` contains a base64-encoded WOFF2 font (~784 bytes) that maps every BMP codepoint to a filled disc (●). The font is embedded as a data URL:

```css
@font-face {
  font-family: "bl-si-censored-disc";
  src: url("data:font/woff2;base64,...") format("woff2");
  font-display: block;
}
```

`font-display: block` prevents the browser from using a system font fallback during font load — the text is invisible (FOIT) until the font loads. Since the font is a data URL (embedded in the CSS), it loads synchronously with the stylesheet.

### CSS generated by `injectRules` (censored):

```css
/* Font face first, so it's defined before usage */
@font-face {
  font-family: "bl-si-censored-disc";
  src: url("data:font/woff2;base64,...") format("woff2");
  font-display: block;
}

h1:not(...EXCLUDE)...,
[data-bl-si-blur]:not([data-bl-si-reveal]) {
  font-family: "bl-si-censored-disc" !important;
  filter: none !important;       /* cancels gaussian fallback */
  user-select: none !important;
}
```

### Media elements in censored mode

Font replacement has no effect on image/video pixel content. Images/videos get:
```css
img:not(...EXCLUDE),
video:not(...EXCLUDE),
...,
img[data-bl-si-blur]:not([data-bl-si-reveal]),
... {
  filter: brightness(0) !important;
  user-select: none !important;
}
```

`brightness(0)` renders the element as solid black regardless of content. This does not hide the element's layout box — the black box occupies the same space as the original image.

### Reveal

```css
[data-bl-si-reveal] {
  filter: none !important;
  visibility: visible !important;
  font-family: unset !important;  /* removes the censored-disc font */
  user-select: auto !important;
}
```

`font-family: unset !important` restores the element's inherited or default font, revealing the original text. `filter: none !important` removes the `brightness(0)` from media elements.

## Frosted Glass (Pick-blur Variant)

Pick-blur frosted mode uses the same SVG filter as blur-all frosted mode. The CSS is injected into `<style id="bl-si-pick-blur-styles">` (a separate style element from the blur-all styles):

```js
if (type === blsi.pick_blur_modes.frosted) {
  ensureSvgFilter(root);  // ensures SVG filter exists in this root
  rules.push(
    '[data-bl-si-pick-blur]:not([data-bl-si-reveal]) { ' +
    'filter: url(#bl-si-frosted-filter) !important; ' +
    'transition: filter var(--bl-si-transition-duration, 150ms) ease !important; ' +
    'user-select: none !important; }'
  );
  rules.push('[data-bl-si-pick-blur][data-bl-si-reveal] { filter: none !important; }');
}
```

Zone overlays in frosted pick-blur mode use `backdrop-filter` from the base `.bl-si-zone-overlay` CSS rule (not the injected frosted rule) — zone overlays are `<div>` elements with no content to filter.

## Color Mode (Pick-blur Only)

Pick-blur color mode covers elements with a flat RGBA color. This is the most visually distinctive pick-blur mode.

```js
if (type === blsi.pick_blur_modes.color) {
  const rgba = _colorToRgba(color);  // hex + opacity → rgba string

  // Regular elements: solid color cover
  rules.push(
    '[data-bl-si-pick-blur]:not(.bl-si-zone-overlay):not([data-bl-si-reveal]) { ' +
    'background-color: ' + rgba + ' !important; ' +
    'color: transparent !important; ' +
    'filter: none !important; ' +
    'user-select: none !important; }'
  );

  // Zone overlays: replace backdrop-filter with flat background
  rules.push(
    '.bl-si-zone-overlay[data-bl-si-pick-blur]:not([data-bl-si-reveal]) { ' +
    'backdrop-filter: none !important; ' +
    '-webkit-backdrop-filter: none !important; ' +
    'background: ' + rgba + ' !important; ' +
    'border: none !important; }'
  );

  // Reveal
  rules.push(
    '[data-bl-si-pick-blur][data-bl-si-reveal] { ' +
    'background-color: transparent !important; ' +
    'color: inherit !important; ' +
    'filter: none !important; ' +
    'user-select: auto !important; }'
  );
}
```

**Color conversion:**
```js
function _colorToRgba(color) {
  if (!color || !color.hex) return 'rgba(0,0,0,1)';
  const hex = color.hex.replace('#', '');
  const r = parseInt(hex.slice(0, 2), 16);
  const g = parseInt(hex.slice(2, 4), 16);
  const b = parseInt(hex.slice(4, 6), 16);
  const a = typeof color.opacity === 'number' ? color.opacity : 1;
  return 'rgba(' + r + ',' + g + ',' + b + ',' + a + ')';
}
```

Input `{ hex: '#FF0000', opacity: 0.8 }` → output `rgba(255,0,0,0.8)`.

**Why zone overlays need a separate rule in color mode:**
Zone overlays are empty `<div>` elements that use `backdrop-filter: blur()` (not `filter: blur()`) to blur content behind them. In color mode, the backdrop-filter must be disabled (otherwise it would show a blur on top of the solid color). The separate rule does both: removes backdrop-filter and sets background to the flat color.

## Starred Mode (PII Only)

Starred mode is exclusive to PII detection. It replaces every character with a 6-arm asterisk glyph (*) using a separate custom WOFF2 font.

### Font

`blsi.Fonts.ASTERISK_FONT_FACE` — a base64-encoded WOFF2 font (~372 bytes) that maps every BMP codepoint to a 6-arm asterisk glyph. Smaller than the disc font (fewer BMP codepoints mapped?), but same mechanism.

### CSS generated by `injectPiiRules` (starred):

```css
@font-face {
  font-family: "bl-si-starred-asterisk";
  src: url("data:font/woff2;base64,...") format("woff2");
  font-display: block;
}

[data-bl-si-pii]:not([data-bl-si-reveal]) {
  font-family: "bl-si-starred-asterisk" !important;
  filter: none !important;
  user-select: none !important;
}

[data-bl-si-reveal] [data-bl-si-pii] {
  filter: none !important;
  font-family: unset !important;
  color: unset !important;
  background-color: unset !important;
  user-select: auto !important;
}
```

PII spans are `<span>` elements wrapping text nodes — they contain text content. Font replacement works correctly because the character glyphs are rendered but replaced by the custom font.
